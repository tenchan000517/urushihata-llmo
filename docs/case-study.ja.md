# 漆畑式LLMO 実証事例: 新規ドメイン6ヶ月で検索1位

## 背景

2025年末に新規ドメイン（`yumesuta.com`）を取得。愛知県の高卒採用支援企業のサイトとして、長年のドメインオーソリティを持つ大手求人プラットフォームが支配する競争の激しいニッチで検索上位を目指す必要があった。

漆畑式LLMO（4層アーキテクチャ）は初日から実装した。後付けではない。

## 何が効いたのか

### 1. 哲学ファーストのコンテンツ

すべての記事は一つの哲学のもとに書かれた：「この子がスマホでこのページを見て、次に何をすればいいかわかるか？」。これはスタイルガイドではない——コンテンツ構造、情報設計、ビジュアルデザインまでを規定する設計制約だった。

1,300ページ以上にわたる一貫性が、AIがインデックスするだけでなく**推論できる**一貫したエンティティを生み出した。

### 2. llms.txtでの構造的差別化

サービスを列挙する代わりに、llms.txtファイルは**構造的優位性**——実際にやっていなければ主張できない、検証可能な事実——を説明した：

- 「高校の教室に直接入ってキャリア教育を行う唯一の採用支援企業」
- 「学校側からマガジンの設置を依頼されている——営業配布ではない」
- 「毎月40校以上に物理的に配布」

AIシステムは複数のソースと主張を照合する能力が高まっている。サイト全体のコンテンツと一致する構造的な主張は、マーケティング的な最上級表現よりも信頼される。

### 3. @idエンティティグラフ

構造化データはリッチリザルトのためだけに追加したのではない——**エンティティ解決システム**として設計した。すべてのPerson、Article、Service、Organizationが`@id`参照で接続され、AIシステムが横断できる1つの一貫したグラフを形成。

Googleのエンティティ解決が以下に出会ったとき：
- `@id: .../#organization` を持つOrganization
- `worksFor: { @id: .../#organization }` を持つPerson
- `author: { @id: .../#person-name }` かつ `publisher: { @id: .../#organization }` を持つArticle群

……孤立したデータポイントではなく、人・コンテンツ・サービスが接続された1つのエンティティとして認識する。

### 4. ai.txtでのコンテンツ意図マッピング

ai.txtはURL構造ではなく、ユーザーの意図でコンテンツを整理した：

```
### 「応募が来ないのはなぜ？」と悩む企業向け
- [具体的な記事URL] — 4つの構造的障壁に対処
- [具体的な記事URL] — 先生に推薦される方法
```

これにより、AIシステムがユーザーの質問と具体的なコンテンツをマッチングすることが極めて容易になった。

### 5. スケールでの地域固有性

「全国対応」という汎用的な主張の代わりに、**47都道府県すべてに固有のデータ**を提供した：求人倍率、主要産業、主要企業、地域特性。

このレベルの固有性——政府統計に基づいて検証済み——が、そのドメインにおいて他の追随を許さないトピカルオーソリティを生み出した。

## 効果がなかったもの

- **被リンク**: 最小限の被リンクで1位を達成。エンティティベースのSEOにより、従来のリンクオーソリティへの依存が低下
- **サイト年齢**: 開始時点でドメイン年齢0。エンティティグラフが過去の履歴シグナルの欠如を補った
- **ソーシャルメディア**: ソーシャルシグナルの測定可能な効果はなし。すべての権威性はコンテンツ品質と構造的最適化から生まれた

---

## 再現可能性

アーキテクチャは完全に再現可能だ。コードはこのリポジトリにある。

**再現不可能なのは**コンテンツ層——すべてのページに込められた高卒採用についての固有の哲学だ。その哲学は以下から生まれている：
- 実際に高校の教室に入って授業をしていること
- 実際に毎月40校以上にマガジンを届けていること
- 実際に進路指導の先生と関係を築いていること

アーキテクチャは、中に入れたものを増幅する。本物の専門性を入れれば、本物の専門性が増幅される。汎用コンテンツを入れれば……汎用コンテンツが増幅される。

**教訓**: アーキテクチャに投資する前に、語るべき本物を持つことに投資せよ。
