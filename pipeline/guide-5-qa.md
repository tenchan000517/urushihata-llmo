# 漆畑式LLMO Pipeline — Step 5: 品質保証 / Quality Assurance

**目的: 完成したページを複数の観点から検証し、修正すべき箇所を報告する。**

**トラック固有のチェック項目はトラック設定ファイル（CONFIG）を参照すること。**

---

## このガイドの使い方

1. 実装済みのページコード（page.tsx + Content.tsx）を受け取る
2. **トラック設定ファイル（CONFIG）を読み、固有のチェック項目を確認する**
3. 以下の共通チェックリスト + CONFIG の固有チェックを全て実行する
4. 不合格の項目は「不合格項目一覧」にまとめて報告する
5. **1つでも不合格なら修正を要求する。**「まあいいか」は許されない

---

## チェック1: 品質基準（CONFIG 参照）

CONFIG の「品質基準」に定義された問いに**全てYes**と言えなければ不合格。

---

## チェック2: コンテンツ品質（共通）

### 行動可能性
- [ ] 読んだターゲットが「次に何をすればいいか」わかるか
- [ ] 手順が具体的か（抽象的な「考え方」で終わっていないか）

### 正確性
- [ ] 全ての数値に出典（調査名・発表元・年度）があるか
- [ ] 推測表現（「一般的に」「よく言われる」「〜でしょう」）が含まれていないか
- [ ] 地域限定の制度やルールを全国共通であるかのように書いていないか
- [ ] 過去の日付を未来形で書いていないか

---

## チェック3: 文体（CONFIG 参照）

CONFIG の「文体設計」に従った口調・トーンになっているか確認。

共通チェック:
- [ ] 数字が文脈に埋め込まれているか
- [ ] 名前表記が統一ルール（姓 名 様、半角スペース）に従っているか

---

## チェック4: デザイン・実装（共通）

### ページ構造
- [ ] `page.tsx`（server component: metadata + StructuredData）と `XxxContent.tsx`（client component: UIロジック）の2ファイル構成か
- [ ] `page.tsx` にレイアウト・スタイルの記述がないか
- [ ] `'use client'` が必要な場合のみ Content.tsx の先頭にあるか

### レイアウト
- [ ] ページ全体の背景色がプロジェクトのデザインシステムに従っているか
- [ ] コンテンツエリアの最大幅・パディングが統一されているか
- [ ] セクション間の余白が一定か
- [ ] カードのスタイル（角丸・背景色）が統一されているか
- [ ] テキストサイズ・色がプロジェクト基準を満たしているか
- [ ] コントラスト比が十分か

### トラック固有のUI
→ CONFIG の「実装チェック」セクションを参照（ヒーロー構造、CTA、ナビゲーション等）

---

## チェック5: 禁止事項（共通）

以下が1つでもあれば不合格。

- [ ] グラデーションが使われていないか
- [ ] 装飾的サイドライン（`border-l-4` 等）が使われていないか（囲み枠はOK）
- [ ] 絵文字アイコンが使われていないか（SVGアイコンライブラリのみ）
- [ ] スワイプUIが使われていないか
- [ ] サブテキストがタイトルと違う色になっていないか
- [ ] SaaS風グラデーション・無意味なシャドウがないか

**プロジェクト固有の禁止事項は CONFIG を参照。**

---

## チェック6: データ登録（CONFIG 参照）

CONFIG の「データ登録」セクションに従い、必要なファイルが更新されているか確認。

共通チェック:
- [ ] `sitemap.ts` にエントリーが追加されているか

---

## チェック7: SEO・構造化データ（共通）

- [ ] `metadata`（title, description, keywords, canonical）が設定されているか
- [ ] `BreadcrumbSchema` が正しいパンくずリストを生成しているか
- [ ] `ArticleSchema` が設定されているか
- [ ] titleの形式がトラックに合っているか

---

## チェック8: 型チェック（共通）

- [ ] TypeScript の型チェックが通るか

---

## チェック9: ユーザー視点の最終確認（CONFIG 参照）

CONFIG の「シチュエーション別チェック」を実行する。

---

## 報告フォーマット

```markdown
# QAチェック結果: [ページタイトル]

## トラック: SEO / LLMO / User

## ステータス: 合格 / 不合格

## 不合格項目一覧
（番号付きリスト。各項目に「チェック番号」「具体的な問題」「修正案」を記載）

1. **[チェック4: レイアウト]** サブテキストの色が基準を満たしていない
   → 基準に合った色に変更

## 合格項目のハイライト
（特に品質が高かった点を1-2個挙げる）
```
