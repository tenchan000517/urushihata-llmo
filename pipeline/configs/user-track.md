# 漆畑式LLMO — User Track CONFIG

**目的**: エンドユーザー（若者・当事者）が自分の課題を考え、行動に移せるコンテンツを制作する
**配置先**: `/[YOUR_USER_CONTENT_PATH]/[slug]`
**ターゲット**: エンドユーザー（当事者）、その保護者、指導者・アドバイザー

---

## ターゲット定義

### 救いたいユーザーが3種類いる

1. 情報不足で「思ったのと違う」と失敗してしまうユーザー
2. 本当はやりたいことがそこにあるのに、情報不足で見つけられず失敗するユーザー
3. 周囲の理解を得られず、やりたいこととは違う選択をするしかないユーザー

この3種類のユーザーを救うことが全てのコンテンツの存在理由。記事に直接書くのではなく、ストーリーの裏側に流れる思想として反映する。

### サイトに来る4種類のユーザー

| # | ユーザー像 | この人が必要としていること |
|---|----------|----------------------|
| 1 | 過去の失敗から再出発する人 | 「前の経験で疲れてるかもしれない。でもこの分野にはこういう居場所がある」 |
| 2 | 情報不足で不安な人 | 「実際の1日ってどんな感じ？」「自分みたいな普通の人でも大丈夫？」 |
| 3 | 周囲に応援されない人 | 保護者・周囲の人が見ても「この分野ならちゃんとしてる」と安心できる情報 |
| 4 | やりたいことを探している人 | 「まだ決まってなくても大丈夫。こんな入り方もある」 |

構成設計のとき、必ずこの4人のうち「このページはどの人に一番刺さるか」を意識する。

### 指導者・アドバイザーも見ている

指導者やアドバイザーは、このコンテンツを見て「この発信元は本当にユーザーのことを考えているか」を姿勢で見抜く。指導者専用のページは要らない。ユーザー向けコンテンツの質そのものが指導者への最大の訴求。

### このページはいつ、どこで、誰と、どうやって見られるのか

| シチュエーション | 求められること |
|---------------|-------------|
| **ユーザーが一人で見る**（夜、自室でスマホ） | 「次に何をすればいいか」が明確。一人でも完結できる行動ステップと「これは専門家に聞いてね」の線引き |
| **指導者と一緒に見る**（相談・指導の時間） | 指導者がこのページを教材として使える構成。「ここを見て」と示せるセクション分け |
| **教育の場で教材として使われる**（集団・プロジェクター） | 「やってみよう」が機能する。全員に当てはまる部分と「自分の場合は？」を考えさせる部分 |
| **周囲の人を説得するための材料として見る** | 数字・出典がある。感想文ではなく事実が並んでいる |
| **指導者が信頼判断の参考にする** | 「この発信元は本当にユーザーのことを考えている」と感じられるか。姿勢で信頼が決まる |

## 品質基準

### 制作思想の4つの問い（全てYesでなければ出す価値がない）

1. 過去の失敗から再出発するユーザーの**次の道**を指し示すものになっているか？
2. 指導者・アドバイザーがこのコンテンツを見て「この発信元は本当にユーザーのことを考えている」と感じ、ユーザーに率先して勧めたくなるか？
3. やりたいことが見つからないユーザーに**希望**を与えるコンテンツになっているか？
4. ユーザーに**寄り添って**いるか？

### 追加基準

- **読んだユーザーがSTEP by STEPで確実に「できる」こと。** これが唯一の品質基準
- **手順と感情の両方を全ページに入れる。** 心のケアは独立ページにしない
- **別の文脈のコンテンツの焼き直しは害悪。** ターゲットにはターゲット固有の仕組みがある

## 文体設計

### 2つの声

| 場所 | 誰の声 | 口調 | 例 |
|------|--------|------|-----|
| 見出し | ユーザーの声 | タメ口 | 「入れるの？」「どんな仕事がある？」「怖いよね」 |
| 本文 | ガイドの声 | 丁寧語（ですます調） | 「入れます。」「大丈夫です。」 |

### なぜこの設計か

ユーザーがタメ口で「入れるの？」と不安を投げかけたとき:
- 同じタメ口で「入れる。」→ 不安を受け止めずに断言している。腹が立つ
- 丁寧語で「入れます。」→ 同じ言葉なのに、温かく受け止めている印象に変わる

### オウム返し禁止

- OK: 「入れるの？」→「入れます。しかも、この業界は人手がまったく足りていません。」
- NG: 「入れるの？」→「入れる。めちゃくちゃ入れる。」

### 心のケアの書き方

- 全てのページに手順と感情の両方を入れる。面接のページなら面接の不安をそのページ内で扱う
- 具体的な不安を言語化する（初めてスーツを着る不安、周りはもう決まっているのに自分は、等）
- 根拠のない「大丈夫」は無責任。「なぜ大丈夫か」を添える

### 数字の翻訳例

- 「求人倍率3.91倍」→「1人に約4社が『来てほしい』と手を挙げている」
- 「内定率99%」→「最終的にほぼ全員が決まっている」
- 「離職率38.4%」→「10人入ったら3年以内に約4人が辞めている」

### 導線の入れ方

- OK: 「相談してみてください。[YOUR_SERVICE]が届いていれば、もっと詳しい情報が載っています。」
- NG: 「[YOUR_SERVICE]なら安心！今すぐお問い合わせ！」

## 構成パターン

### パターンUser-1: 行動ガイド型

「もう動くと決めた人」が来る場所。具体的な手順書。

| 順序 | セクション | 目的 |
|------|----------|------|
| 1 | ヒーロー | タメ口の問いかけ + サブテキスト（丁寧語）+ キャラクター（使用する場合） |
| 2 | 誤解/共感 | 「知らなくて当然」「不安で当然」と寄り添う |
| 3-5 | 本文セクション群 | STEP by STEPの行動手順 |
| 6 | 心のケアまとめ | テーマ色 + 白テキストのまとめボックス |
| 7 | CTA | 関連サービスへの自然な誘導 |
| 8 | 出典 + ナビ | 出典一覧 + 前/次ページ + カテゴリナビ |

### パターンUser-2: 分野探索型

「この分野ってどんなところ？」を知りたい人が来る場所。

| 順序 | セクション | 目的 |
|------|----------|------|
| 1 | ヒーロー | 誤解やイメージに触れつつ「覗いてみよう」と誘う |
| 2 | 思い込み3つ | ユーザーが持ちがちな誤解を提示 |
| 3 | 入れるの？ | 最大の不安に直接回答 |
| 4 | どんな仕事がある？ | 4職種を紹介 |
| 5 | ある1日の流れ | 代表的な1職種のタイムライン。帰り道の気持ちを必ず入れる |
| 6 | 身につくスキル | 5つのスキル |
| 7 | キャリアステップ | 入門〜10年超の4ステップ |
| 8 | この仕事のリアル | 離職率を正直に。見極めポイント4つ |
| 9 | やっててよかった | リアルの直後に必ず希望を見せる |
| 10 | スポットライト | 実際の事例があれば表示 |
| 11 | CTA + 出典 + ナビ | |

**重要:** 「リアル」(8)と「やっててよかった」(9)は必ずセットで設計する。

### パターンUser-3: 自己発見型

「やりたいことがわからない」人が来る場所。

| 順序 | セクション | 目的 |
|------|----------|------|
| 1 | ヒーロー | タメ口のタイトル（手法の名前は使わない） |
| 2 | なぜこれが大事？ | 「やってみたい」と思わせる |
| 3 | やってみよう | 具体的なやり方をステップで |
| 4 | やってみるとこうなる | 架空のユーザー2-3人の例 |
| 5 | 気づきのポイント | 3つの重要な気づき |
| 6 | キャリアとのつながり | 自己分析の結果が選択にどう繋がるか |
| 7 | メソッド背景 | 学術的根拠を軽く |
| 8 | CTA + 出典 + ナビ | |

## CTA・導線

### 使用コンポーネント

- **ユーザー向けCTA** — メインサービスへの誘導
- **カテゴリナビ** — 全ページ末尾に必須（例: STEP 1/2/3 のカード）
- **前/次ページナビ** — 関連ページへのカードリンク

### キャラクターシステム（例）

ユーザーの親しみやすさと感情移入を促すために、ガイドキャラクターを使うことができる。

| キャラ | 担当領域 | 用途 |
|--------|---------|------|
| キャラA | 体を使う・ものづくり系 | 該当分野のガイド |
| キャラB | 知識・技術系 | 該当分野のガイド |
| キャラC | ケア・クリエイティブ系 | 該当分野のガイド |
| キャラD | 接客・対人サービス系 | 該当分野のガイド |

| ポーズ | 使う場面 |
|--------|---------|
| thinking | ヒーローの共感・疑問 |
| search | スクロール示唆 |
| guide | 心のケア・案内セクション |

キャラクターの具体的なデザイン・ファイル名はプロジェクト固有の設定に従う。

## URL・配置

- **パス**: `/[YOUR_USER_CONTENT_PATH]/[slug]`（分野別: `/[YOUR_USER_CONTENT_PATH]/industries/[slug]`、職種別: `/[YOUR_USER_CONTENT_PATH]/jobs/[slug]`）
- **パンくず**: ホーム → [YOUR_USER_SECTION_NAME] → [ページ名]
- **title形式**: `ページタイトル | [YOUR_USER_SECTION_NAME] | [YOUR_SITE_NAME]`
- **canonical**: `https://[YOUR_DOMAIN]/[YOUR_USER_CONTENT_PATH]/[slug]`

## 実装チェック（QA用）

### ヒーロー
- [ ] ヘッダー内に配置されているか
- [ ] 戻るリンクがあるか
- [ ] カテゴリバッジがあるか
- [ ] h1がタメ口の問いかけか
- [ ] サブテキストが丁寧語で、h1と同じ色系統か
- [ ] キャラクター（使用する場合）が正しいポーズで表示されているか

### CTA・ナビ
- [ ] ユーザー向けCTA コンポーネントが配置されているか
- [ ] カテゴリナビがページ末尾にあるか
- [ ] 前/次ページナビがあるか

### 心のケア
- [ ] 手順だけでなく感情・不安への対処が含まれているか
- [ ] 心のケアまとめボックスがあるか
- [ ] 「大丈夫」に根拠が添えられているか

## シチュエーション別チェック（QA用）

| シチュエーション | チェック |
|---------------|---------|
| ユーザーが夜、一人でスマホで見る | 次に何をすればいいか明確か。曖昧なまま終わっていないか |
| 指導者が相談の時間にユーザーと一緒に見る | 教材として使えるセクション分けになっているか |
| 周囲の人を説得するために見せる | 数字・出典があり、信頼できる見た目か |
| 指導者が「この発信元を信頼できるか」判断する | 姿勢が伝わるか。ユーザーの立場に立てているか |
