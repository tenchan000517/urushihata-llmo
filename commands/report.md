# 活動報告の作成

ユーザーから共有された **事実情報** に基づいて、活動報告（ニュース記事）を作成する。

---

## 最重要ルール

### 事実厳守の原則

- **共有された事実のみ** を記事にする。推測・補足・背景説明を勝手に追加しない
- ユーザーが言及していない情報、期待、展望、意気込みなどを **絶対に捏造しない**
- 「文章として成り立たせる」ための最低限の接続・敬語・挨拶文は許可する

### 許可される追加

```
OK: 事実を丁寧語に整える
OK: 末尾に「引き続きよろしくお願いいたします」程度の挨拶
OK: 事実を読みやすく段落分けする
```

### 禁止される追加

```
NG: ユーザーが言っていない「今後の展望」「目標」「意気込み」を追加
NG: 「業界をリードする」「革新的な」等の装飾的な形容詞
NG: 言及されていない背景情報や文脈の捏造
NG: 事実にない数字や日付の補完
```

---

## 入力

ユーザーから以下が共有される（全て揃わない場合もある）:

1. **画像パス**: 使用する画像ファイル（任意）
2. **テーマ/タイトル**: 報告の主題
3. **事実**: 報告すべき内容（箇条書きやメモ形式）
4. **カテゴリ**: おしらせ / プレスリリース / 採用情報 / イベント

---

## 出力

### Step 1: 下書きの提示

まず Markdown の下書きをユーザーにテキストとして提示する。**ファイルには書き込まない。**

確認事項:
- 「事実と異なる記述はありませんか？」
- 「追加・削除したい内容はありますか？」

### Step 2: 承認後にファイル作成

ユーザーの承認を得てから:
1. `[YOUR_NEWS_PATH]/{slug}.md` にMarkdownファイルを作成
2. 必要に応じて `sitemap.ts`、`ai.txt`、`llms.txt` を更新

---

## 文体ガイドライン

- **敬体**（です・ます調）
- 簡潔で読みやすい文章
- 1段落は2〜3文程度
- 過度な装飾を避け、事実を淡々と伝える
