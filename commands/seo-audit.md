# SEO/LLMO 技術監査

現在のプロジェクトに対して、包括的なSEO/LLMO技術監査を実行する。

---

## 監査手順

### Phase 1: プロジェクト検出

```
- フレームワーク検出: Next.js / Nuxt / Gatsby / plain HTML 等
- package.json, next.config.* 等を確認
- src/ or pages/ ディレクトリ構成を把握
```

### Phase 2: 並列監査（5エージェント同時起動）

以下の5領域を Agent ツールで並列実行する（`subagent_type: "Explore"`）:

| Agent | 監査領域 | チェック内容 |
|-------|---------|-------------|
| 1 | メタデータ | title/description/OG画像のカバレッジ、canonical URL |
| 2 | 構造化データ | Schema.org実装、@id相互参照、Organization集中管理 |
| 3 | LLMO対策 | robots.txt AIクローラー許可、ai.txt/llms.txt品質、サイトマップ網羅性 |
| 4 | パフォーマンス | Image最適化、RSC/Client比率、フォント、コード分割 |
| 5 | セキュリティ | ヘッダー設定、環境変数、APIルート検証 |

### Phase 2.5: クロス検証（必須）

**エージェントの出力をそのまま信用しない。**

1. 「未カバー」報告のうち5件を実際にファイルを読んで確認
2. 数値の分母と分子の根拠を確認
3. 環境固有の誤検知を除外

### Phase 3: レポート生成

```markdown

| 構造化データ | X/10 | ✅/⚠️/❌ |
| LLMO対策 | X/10 | ✅/⚠️/❌ |
| パフォーマンス | X/10 | ✅/⚠️/❌ |

```

---

## 注意事項

- **読み取り専用**: コードの変更を行わない（調査・レポートのみ）
- **プロジェクト汎用**: 特定プロジェクトへのハードコードなし
