# サイト設定テンプレート / Site Configuration Template

このファイルを複製して、自社サイトの分析設定を定義する。
分析フレームワーク（`analysis-framework.md`）はこの設定に基づいて分析を行う。

---

## ビジネスファネル定義

自社のコンバージョンまでの導線を段階に分ける。

```
[認知] → [流入] → [関心] → [行動] → [コンバージョン]
```

| 段階 | 定義 | 計測指標 | 該当ページ |
|------|------|---------|----------|
| 認知 | 検索結果に表示される | SC表示回数 | 全ページ |
| 流入 | サイトに来る | GA4セッション | 全ページ |
| 関心 | サービスを見る | GA4ページビュー | [サービスページURLを列挙] |
| 行動 | 問い合わせに進む | GA4ページビュー | [問い合わせページURL] |
| CV | 問い合わせ送信 | GA4イベント | [イベント名: form_submit等] |

**最大の落差 = ボトルネック**。ここに集中して改善する。

---

## ターゲット区分

サイトに複数のターゲットがいる場合、区別して分析する。

| ターゲット | 説明 | 代表的な流入ページ | 成功指標 |
|-----------|------|------------------|---------|
| [B2Bターゲット名] | [説明] | [URL群] | 問い合わせ |
| [消費者ターゲット名] | [説明] | [URL群] | エンゲージメント |

**注意**: B2Bのコンバージョンが目的なのに、消費者向けページの改善をACTIONにしない。ターゲット区分を間違えない。

## ページグループ

GA4のページ別データを分析する際のグループ分け。

| グループ名 | URLパターン | 役割 |
|-----------|-----------|------|
| [グループ1] | /blog/* | コンテンツハブ |
| [グループ2] | /services/* | サービス紹介 |
| [グループ3] | /regions/* | 地域ページ |
| [グループ4] | / | トップページ |
| [グループ5] | /contact | コンバージョンページ |
