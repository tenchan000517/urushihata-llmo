# 漆畑式LLMO — 分析フレームワーク / Analysis Framework

データ取得後、以下のフレームワークに沿って分析を行う。
**サイト固有の設定は別途定義された設定ファイルを参照すること。**

---

## 分析の流れ

```
1. ビジネスファネル分析（最重要・最初にやる）
   ↓ 「どこが詰まっているか」を特定
2. 鳥の目（全体俯瞰）
   ↓ 成長しているか、構造は健全か
3. 虫の目（焦点・深掘り）
   ↓ ファネルの詰まりの原因を深掘り
4. 魚の目（時流・変化）
   ↓ 環境変化・新しい兆候
5. ACTION決定（1〜3個に絞る）
   → 必ずファネルの詰まりに対応するアクションにする
```

**鉄則: ACTIONはファネルの詰まりに直結するものだけ。「改善できそう」ではなく「ビジネスに効く」で選ぶ。**

---

## 1. ビジネスファネル分析

### 手順

プロジェクト固有のファネル定義に従い、上から順に今月の数値を埋める。

ファネルの設計はビジネスモデルによって異なるが、一般的には以下の段階を含む:

```
認知（検索表示・SNS露出等）
  ↓
流入（サイト訪問）
  ↓
関心（回遊・コンテンツ閲覧）
  ↓
行動（問い合わせ・申込・購入等）
```

各プロジェクトで具体的な段階名と計測指標を定義すること。

### 分析ポイント

- **最大の落差はどこか？** — 段階間の転換率を計算し、最も落ちている箇所を特定
- **前月比でどの段階が変わったか？** — 改善/悪化した段階を特定
- **開発者アクセスの除外** — 自分のアクセスが含まれていないか確認（特にコンバージョン系）

### 出力

```markdown
## ビジネスファネル

| 段階 | 今月 | 前月比 | 転換率 |
|------|------|--------|--------|
| [段階1] | ○ | — | — |
| [段階2] | ○ | — | ○→○ = ○% |
| [段階3] | ○ | — | ○→○ = ○% |
| [段階4] | ○ | — | ○→○ = ○% |

**ボトルネック**: [最も転換率が低い段階]
```

---

## 2. 鳥の目（全体俯瞰）— サイトは成長しているか？

### 見る指標

| 指標 | データソース | 問い |
|------|------------|------|
| アクティブユーザー数 | GA4 | 前月比で成長しているか？ |
| 新規率 | GA4 | 新しいユーザーが来ているか？ |
| 回遊率（PV/セッション） | GA4 | コンテンツに興味を持っているか？ |
| 直帰率 | GA4 | 期待外れで帰っていないか？ |
| 流入チャネル比率 | GA4 | 特定チャネルに依存しすぎていないか？ |
| デバイス構成 | GA4 | ターゲット層と一致しているか？ |

### 分析ポイント

- **デバイス構成はターゲットと整合するか？** — B2BならPC優位、若者ならモバイル優位が正常
- **流入チャネルの偏り** — 1チャネル70%以上は危険。分散が健全
- **Bing/Yahoo比率** — 日本では通常5-10%。異常に高い場合は理由を考える
- **前月比** — 2回目以降はトレンドを重視。単月の絶対値より変化率

### 出力

トラフィック概況テーブル + 流入チャネル構成テーブル + デバイス構成テーブル + 1-2行の所感

---

## 3. 虫の目（焦点・深掘り）— ファネルの詰まりの原因は？

### 見る指標

ファネル分析で特定したボトルネックに応じて、深掘り対象を選ぶ。

#### パターンA: 流入が少ない場合（ファネル上部の詰まり）

| 指標 | データソース | 問い |
|------|------------|------|
| ターゲットページの表示回数 | Search Console | そもそも検索結果に出ているか？ |
| ターゲットページの順位 | Search Console | 順位が低すぎないか？ |
| ターゲットページのCTR | Search Console | 順位の割にクリックされていないか？ |
| ターゲットKWの順位変動 | SC + ベースライン | 前回から改善/悪化したか？ |

#### パターンB: 流入はあるが次段階に進まない場合（ファネル中間の詰まり）

| 指標 | データソース | 問い |
|------|------------|------|
| 該当ページの直帰率 | GA4 | 読んで満足して帰っていないか？ |
| 該当ページの滞在時間 | GA4 | ちゃんと読まれているか？ |
| 次ページへの遷移 | GA4 | どこに（どこにも）行っているか？ |
| CTA/導線の存在 | ページ確認 | 次の行動への誘導があるか？ |

#### パターンC: 最終段階で脱落する場合（ファネル下部の詰まり）

| 指標 | データソース | 問い |
|------|------------|------|
| フォーム到達数 | GA4 | フォームまで来ているか？ |
| フォーム送信数 | GA4 | 来たのに送信していないか？ |
| フォームの項目数/UX | ページ確認 | ハードルが高すぎないか？ |

### 出力

ボトルネックに対応する深掘りテーブル + 「即効性ランキング」（表示回数×改善余地でソート） + 原因の仮説

---

## 4. 魚の目（時流・変化）— 潮目が変わっていないか？

### 見る指標

| 指標 | データソース | 問い |
|------|------------|------|
| AI経由の流入推移 | GA4 referral | LLMO施策は効いているか？ |
| 新たにランクインしたKW | Search Console | 新しい検索ニーズが生まれていないか？ |
| 季節性 | 業界知識 | 今は繁忙期か閑散期か？次月はどうか？ |
| サブドメイン/外部資産 | Search Console | 本体以外の資産が成長していないか？ |
| 競合の動き | 任意 | 同じKWで新しい競合が出現していないか？ |

### 分析ポイント

- **AI流入は「0か、0でないか」が重要** — 絶対数よりも存在の有無。0→1が最大の変化
- **季節性を無視しない** — 前月比の悪化が季節要因なら問題ない
- **新KWは「意図しないもの」に注目** — 狙っていないKWでランクインしていたら、新しい需要の兆候

### 出力

AI流入テーブル + 新規KWリスト + 季節性の所感 + 1-2行の「潮目判定」

---

## 5. ACTION決定

### ルール

1. **最大3個**に絞る（多すぎると何もやらない）
2. **ファネルのボトルネックに直結する**ものだけ選ぶ
3. 各ACTIONに以下を明記する:
   - 現状の数値
   - 目標の数値
   - 具体的にやること
   - 期待効果
   - 工数の目安

### ACTIONの優先度判断

```
即効性（すぐ効果が出る）× インパクト（ファネルへの影響度）

高即効・高インパクト → 最優先（title/desc改善、導線追加）
高即効・低インパクト → 余力があれば
低即効・高インパクト → 今月着手、来月効果確認
低即効・低インパクト → やらない
```

### 出力

```markdown
## 今月のACTION

### ACTION 1: [タイトル]（即効性: S/A/B）
- **ファネル段階**: [どの段階の改善か]
- **現状**: [数値]
- **目標**: [数値]
- **やること**: [具体的な作業]
- **期待効果**: [定量的に]
- **工数**: [時間の目安]
```

## 前月比トラッキング（2回目以降）

前月の分析レポートが存在する場合、以下を追加:

1. **ファネル各段階の前月比** — 改善/悪化をファネル図で可視化
2. **前月ACTIONの効果検証** — 実施したACTIONの結果を数値で確認
3. **KWベースラインとの比較** — ベースラインドキュメントとの差分

```markdown
## 前月ACTIONの振り返り

| ACTION | 実施状況 | 結果 | 判定 |
|--------|---------|------|------|
| [前月のACTION1] | 完了/未着手 | [数値変化] | 効果あり/なし/判定不能 |
```
