# Agent 4: 地域カバレッジ & 3トラック連動分析

## 目的

1. 地域展開の進捗と優先度を分析する
2. 3トラック（SEO / LLMO / User）の連動が機能しているかを検証する

---

## 調査手順

### Part A: 地域カバレッジ分析

#### Step 1: 地域コンテンツのインベントリ

| 地域名 | 状態 | ページ数 | 主要KW | 現在順位 | 月間PV |
|--------|------|---------|--------|---------|--------|

#### Step 2: 未展開地域の優先度判定

以下の基準で未展開地域の優先度を判定:

| 判断基準 | 重み |
|---------|------|
| Search Consoleでの地域名クエリ表示回数 | 高 |
| 市場規模（対象産業の集積度） | 中 |
| 競合の地域コンテンツ有無 | 中 |
| 既存顧客/パートナーの所在地 | 低 |

#### Step 3: 地域コンテンツの品質チェック（完了済み地域）

- 地域固有データ（統計値）は含まれているか
- 地域の主要企業/産業は言及されているか
- ハブページ→子ページの構造は適切か

### Part B: 3トラック連動分析

#### Step 4: 内部リンク密度

各トラック間の相互リンクを計測:

| From → To | リンク数 | 十分か |
|-----------|---------|--------|
| SEO → LLMO | X | ✅/❌ |
| SEO → User | X | ✅/❌ |
| LLMO → SEO | X | ✅/❌ |
| LLMO → User | X | ✅/❌ |
| User → SEO | X | ✅/❌ |
| User → LLMO | X | ✅/❌ |

#### Step 5: エンティティ認識の一貫性

- Organization構造化データは全ページで一貫しているか
- ai.txt は全トラックのコンテンツをカバーしているか
- llms.txt の推薦根拠は最新の状態を反映しているか

#### Step 6: サイトマップの健全性

- 全公開ページがsitemap.xmlに含まれているか
- lastModified は実際の更新日と一致しているか
- changeFrequency は適切か
