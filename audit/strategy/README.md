# 漆畑式LLMO — コンテンツ戦略監査 / Content Strategy Audit

## 根本方針 / Core Principle

### コンテンツに「完成」はない

3トラックそれぞれが**常に成長し続ける**。「これで十分」という判断をしない。
監査の目的は「完成度の評価」ではなく、**「次に何を作るべきか」の特定**。

### 戦略の全体像

```
ユーザーの悩み → あなたのドメインが解決策の一つ → あなたの組織が推薦される

この導線を成立させるために、同時並行で全方位に成長し続ける:
- SEO Track: あらゆるKWで情報覇権を取る
- LLMO Track: AIへの悩み相談に対応し続ける
- User Track: エンドユーザーを集め続ける
- Regional: 地域SEOを制覇する（該当する場合）
```

### 3トラックの連動設計

```
LLMO Track: 悩み1本で全国カバー → 地域別に作る必要なし
User Track: 職種/業界1本で全国カバー → 地域別に作る必要なし
SEO Track (Regional): ここだけ地域別に必要 → テンプレート横展開で効率化

→ LLMO + User が全国共通だからこそ、SEOの地域展開に集中できる
→ 3トラックが連動して、1+1+1 が 10 になる
```

## 実行手順 / Execution Flow

### Phase 1: パフォーマンスデータの準備

**鉄則: 実績データなしの主観的な「このテーマが必要」判断はNG。**

必要なデータ:
- GA4 直近30日のトラフィックデータ
- Search Console 直近30日のクエリ・ページデータ
- KWベースライン（順位追跡データ）

データ取得には [`tools/analytics/`](../../tools/analytics/) のスクリプトを使用。

### Phase 2: 並列コンテンツ分析（4エージェント同時起動）

4つの領域を並列で分析する:

| Agent | 分析対象 | ガイド |
|-------|---------|-------|
| 1 | LLMO/AI引用コンテンツ | [`agents/agent1-llmo-content.md`](agents/agent1-llmo-content.md) |
| 2 | SEO権威性 | [`agents/agent2-seo-authority.md`](agents/agent2-seo-authority.md) |
| 3 | ユーザー向けコンテンツ | [`agents/agent3-user-content.md`](agents/agent3-user-content.md) |
| 4 | 地域カバレッジ＋連動 | [`agents/agent4-regional-synergy.md`](agents/agent4-regional-synergy.md) |

### Phase 3: 統合レポート生成

**重要: スコアは「完成度」ではなく「成長フェーズ」を示す。10/10は存在しない。**

### Phase 4: AI推薦テスト（最終成功指標）

**これが最終的な成功指標。全ての施策はこのテストの結果を変えるために行う。**

```markdown

| AI | 質問 | あなたの記事が引用されたか |
|----|------|-------------------------|

## テスト2: サービス提供者としての推薦（最重要）
| AI | 質問 | あなたの組織が推薦されたか | 何番目か |

|----|------|-------------------------------|
```

## 注意事項

- **読み取り専用**: コードの変更を行わない（調査・レポートのみ）
- **データ駆動**: 全ての提案にGA4/SC/KWの数値を根拠として付ける
- **主観排除**: 「あると良さそう」「一般的に必要」という提案はNG
- **「完成」判定をしない**: 常に「次に何を作るべきか」を提案する
- **既存スキルとの連携**: 制作は各トラックのパイプラインに委譲する
